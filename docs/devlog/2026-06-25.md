## 一、BP_AncientGuardian 受击硬直

### 1. 目的：让 Boss 受击后暂停 AI 行为

在 `BP_AncientGuardian` 的 `Event Tick` 中，已加入 `IsHitStunned` 判断。

当前逻辑意图：

```text
Event Tick
-> Branch: NOT IsDead
-> Branch: IsHitStunned
   True: 空置，不继续追击和攻击
   False: 继续检测玩家、追击、攻击和 Boss 血条显示逻辑
```

校对结论：

- `IsHitStunned = true` 时，Boss 不继续执行追击和攻击逻辑；
- `IsHitStunned = false` 时，Boss 恢复原有 AI 行为；
- 该结构可以保证受击硬直期间 Boss 不会继续攻击玩家。

### 2. 目的：Boss 受击时进入硬直并自动恢复

在 `BP_AncientGuardian` 的 `Event AnyDamage` 非死亡分支中，已加入硬直状态。

当前逻辑：

```text
Event AnyDamage
-> Branch: NOT IsDead
-> Set CurrentHP = CurrentHP - Damage
-> UpdateBossHealthOnHUD(CurrentHP, MaxHP)
-> PrintString("Enemy Hit")
-> Branch: CurrentHP <= 0
   True: 死亡流程
   False:
      Set IsHitStunned = true
      Stop Movement Immediately
      Launch Character
      Retriggerable Delay(HitStunTime)
      Set IsHitStunned = false
```

校对结论：

- 硬直只在 Boss 没死时触发；
- Boss 死亡时不会进入硬直恢复分支；
- `Retriggerable Delay` 可以让连续命中刷新硬直时间；
- `HitStunTime` 当前默认值为 `0.25`。

## 二、受击停止移动

### 1. 目的：避免 Boss 受击后继续沿旧 AI MoveTo 指令滑动

在 `Set IsHitStunned = true` 后，插入：

```text
Stop Movement Immediately
```

中文节点名：

```text
立即停止移动
```

英文节点名：

```text
Stop Movement Immediately
```

Target：

```text
Character Movement
```

校对结论：

- `Stop Movement Immediately` 接在硬直开始后、击退前；
- Target 接的是 Boss 的 `Character Movement` 组件；
- 该节点可以清除 Boss 当前移动速度；
- 后续 `Event Tick` 在硬直结束后仍能重新驱动 Boss 追击玩家。

## 三、Boss 受击击退

### 1. 目的：让玩家命中 Boss 后获得明显反馈

在 `Stop Movement Immediately` 后加入：

```text
Launch Character
```

中文节点名：

```text
发射角色
```

英文节点名：

```text
Launch Character
```

Target：

```text
self
```

校对结论：

- `BP_AncientGuardian` 的父类是 `Character`；
- 因此可以直接对 `self` 调用 `Launch Character`；
- `XYOverride` 已勾选；
- `ZOverride` 已勾选。

### 2. 目的：计算 Boss 远离玩家的击退方向

当前击退方向计算：

```text
BossLocation - PlayerLocation
-> Normalize
-> * HitKnockbackStrength
```

对应蓝图节点：

```text
Get Actor Location(self)
Get Actor Location(PlayerRef)
Vector - Vector
Normalize
Vector * Float
```

校对结论：

- 减法方向是 `Boss位置 - 玩家位置`；
- 因此得到的是 Boss 远离玩家的方向；
- 如果后续出现 Boss 朝玩家方向弹回的问题，需要对调 `Vector - Vector` 两个输入。

### 3. 目的：给击退增加轻微向上分量

当前向上击退使用：

```text
Make Vector
X = 0
Y = 0
Z = HitKnockbackUpStrength
```

然后与水平击退向量相加：

```text
(方向 * HitKnockbackStrength) + (0, 0, HitKnockbackUpStrength)
-> Launch Velocity
```

校对结论：

- 水平击退和向上击退已经合并后接入 `Launch Character.Launch Velocity`；
- 当前向上分量用于让受击反馈更明显，但不会把 Boss 弹得过高。

## 四、变量整理

### 1. 目的：避免击退数值散落在节点常量中

最初击退测试时，蓝图里曾直接使用手填常量：

```text
水平击退：450.0
向上击退：80.0
```

后续已整理为变量连接：

| 变量名 | 类型 | 当前默认值 | 作用 |
|---|---|---:|---|
| `HitStunTime` | Float | 0.25 | Boss 受击硬直持续时间 |
| `HitKnockbackStrength` | Float | 450 | Boss 水平击退强度 |
| `HitKnockbackUpStrength` | Float | 80 | Boss 向上击退强度 |
| `IsHitStunned` | Boolean | false | Boss 是否处于受击硬直 |

校对结论：

- `HitKnockbackStrength` 已接到 `Vector * Float`；
- `HitKnockbackUpStrength` 已接到 `Make Vector.Z`；
- `HitStunTime` 已接到 `Retriggerable Delay.Duration`；
- 后续调参只需要修改变量默认值，不需要再进节点里找手填常量。

## 八、下一步建议

下一步建议继续做 v0.6 的基础打击反馈，而不是马上增加新系统。

推荐顺序：

1. 先做命中特效和命中音效；
2. 再做轻微镜头抖动；
3. 最后再回头调玩家攻击滑步和攻击期间移动速度。

原因：

- 当前连击、命中检测、受击硬直、击退已经形成闭环；
- 命中特效和音效能最快提升“打中了”的反馈；
- 镜头抖动需要较克制，放在特效和音效之后更容易调；
- 玩家攻击滑步属于动作手感细调，适合在反馈链路稳定后处理。
