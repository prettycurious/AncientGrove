## 六、玩家攻击漂移问题排查记录

### 1. 目的：确认攻击漂移是否来自移动输入

继续排查玩家普通攻击时的漂移问题。当前现象是：玩家不按 `W/A/S/D`，只点击鼠标左键攻击时，角色也会向前平移一段距离。

这说明问题不是普通移动输入残留造成的，而更可能来自攻击动画 `MM_Attack_01` 的根运动（Root Motion）或动画本身的位移。

### 2. 目的：确认攻击蓝图当前接线状态

根据截图校对，`BP_PlayerCharacter` 中 `IA_Attack` 当前仍是基础攻击流程：

```text
EnhancedInputAction IA_Attack
-> Started
-> Branch(NOT IsAttacking)
-> Set IsAttacking = true
-> Play Anim Montage(AM_PlayerAttack)
-> Delay(0.25)
-> Sphere Trace By Channel
-> Break Hit Result
-> Branch
```

命中分支当前流程：

```text
Branch True
-> Apply Damage
-> Delay(0.75)
-> Set IsAttacking = false
```

未命中分支当前流程：

```text
Branch False
-> Delay(0.75)
-> Set IsAttacking = false
```

当前截图中尚未形成最终可用的攻击位移修复方案，未保留 `Stop Movement Immediately`、`Set Ignore Move Input` 或 `Set Anim Root Motion Translation Scale` 作为确认后的正式接线。

### 3. 目的：确认动画资源和动画蓝图状态

本轮重点检查了 `AM_PlayerAttack` 引用的源动画：

```text
MM_Attack_01
```

已确认 `BP_PlayerCharacter` 的 Mesh 使用：

```text
动画模式（Animation Mode）：Use Animation Blueprint
动画类（Anim Class）：ABP_Unarmed_C
```

已确认 `ABP_Unarmed` 的根运动模式为：

```text
根运动模式（Root Motion Mode）：Root Motion from Montages
```

这表示：当攻击通过动画蒙太奇（Animation Montage）播放，并且 `MM_Attack_01` 启用根运动（Enable Root Motion）时，Root Motion 会被蒙太奇应用到角色上。

### 4. 目的：记录已尝试但未成功的方向

本轮尝试围绕 `MM_Attack_01` 的根运动（Root Motion）设置进行排查。

尝试一：

```text
启用根骨骼运动（Enable Root Motion）：开启
根运动根锁定（Root Motion Root Lock）：Ref Pose
强制根锁定（Force Root Lock）：关闭
使用标准化根运动范围（Use Normalized Root Motion Scale）：开启
```

结果：攻击时可以看到角色向前位移，但表现更像整体平移，距离和手感不理想。

尝试二：

```text
启用根骨骼运动（Enable Root Motion）：关闭
强制根锁定（Force Root Lock）：开启
```

结果：角色不再平移，但攻击动作里的前跨步也基本消失，动作观感变得不自然。

尝试三：

```text
启用根骨骼运动（Enable Root Motion）：关闭
强制根锁定（Force Root Lock）：关闭
```

结果：动画里能看到脚步向前跨，但由于胶囊体（Capsule）没有真实移动，攻击结束后角色会回到原来的位置，出现“弹回原位”的观感。

### 5. 目的：记录当前结论

本轮没有成功修复玩家攻击漂移问题。今天针对攻击漂移的修复尝试已取消，没有作为最终方案保留。

当前目标更明确：希望达到方案 A：

```text
攻击时胶囊体（Capsule）真实向前小跨一步，攻击结束后停在新位置。
```

后续应继续寻找一种可控的短距离攻击前移方案，而不是在“完全原地”和“完整 Root Motion 平移过远”之间来回切换。

### 6. 目的：记录下一步建议

下一轮建议优先从以下方向继续：

1. 继续保留 `ABP_Unarmed` 的 `Root Motion from Montages`，确认蒙太奇根运动链路正确；
2. 尝试在 `BP_PlayerCharacter` 攻击开始前加入 `Set Anim Root Motion Translation Scale`，用 `0.2`、`0.3`、`0.4` 逐步校对位移距离；
3. 如果 `Set Anim Root Motion Translation Scale` 在当前蓝图中不方便使用，则优先测试 `MM_Attack_02`、`MM_Attack_03` 是否有更适合原地或短距离前跨的攻击动画；
4. 测试时保持控制台命令 `show collision` 开启，明确观察胶囊体（Capsule）是否移动、是否停在新位置；
5. 在确认方案前，不再把攻击动画 Root Motion 设置反复作为最终保存方案。
