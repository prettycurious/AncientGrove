## 一、UE 5.8 Camera Shake 类校对

### 1. 目的：确认 UE 5.8 中应优先使用的 Camera Shake 方案

最初曾考虑使用旧的 `MatineeCameraShake` 方案。后续结合本机 UE 5.8 源码和 Unreal MCP 重新校对后，确认当前项目优先使用：

```text
CameraShakeBase
```

而不是：

```text
MatineeCameraShake
```

校对结论：

- `CameraShakeBase` 在 UE 5.8 中可见；
- `WaveOscillatorCameraShakePattern` 在 UE 5.8 中可见；
- `PerlinNoiseCameraShakePattern` 在 UE 5.8 中可见；
- `MatineeCameraShake` 在 UE 5.8 中属于旧兼容方向，不作为本次首选。

### 2. 目的：通过 Unreal MCP 确认类可见性

Unreal MCP 查询结果确认存在：

```text
/Script/Engine.CameraShakeBase
/Script/EngineCameras.WaveOscillatorCameraShakePattern
/Script/EngineCameras.PerlinNoiseCameraShakePattern
/Script/EngineCameras.LegacyCameraShake
```

因此本次采用：

```text
CameraShakeBase
-> Wave Oscillator Camera Shake Pattern
```

## 七、下一步建议

下一步建议继续补足攻击过程反馈：

```text
3 段攻击挥击音效
```

推荐原因：

- 当前命中时已有音效、特效和镜头抖动；
- 但空挥或攻击挥出的瞬间仍然缺少空气挥击声；
- 挥击音效不影响命中检测、伤害、Boss 逻辑，风险较低；
- 做完挥击音效后，再进入攻击滑步、Hit Stop 或方向性受击动画会更稳。

建议顺序：

1. 为 `Attack_1`、`Attack_2`、`Attack_3` 准备 3 个轻量挥击音效；
2. 在 `AM_Player_Attack_Combo` 中加入挥击 Notify；
3. 在 `ABP_Unarmed` 或 `BP_PlayerCharacter` 中播放对应挥击音效；
4. 测试空挥也能听到挥击声，命中时则同时听到挥击声和命中声。
