## 五、攻击期间角色转向接入与修复

### 1. 目的：记录先接入攻击转向，再修复 Root Motion 下不生效的问题

本步骤的实际顺序：

```text
先增加攻击期间转向控制；
再排查并修复镜头转向后角色没有平滑转向的问题。
```

先接入的攻击转向结构：

| 阶段 | 处理方式 |
|---|---|
| 攻击起手 | `SetAttackTurnEnabled(true, AttackStartTurnRateYaw)`，允许角色朝镜头方向平滑转向 |
| 命中阶段 | `AnimNotify_AttackTurnLock` 调用 `SetAttackTurnEnabled(false, 0)`，锁定攻击朝向 |
| 收招阶段 | `AnimNotify_AttackTurnRecover` 调用 `SetAttackTurnEnabled(true, AttackRecoverTurnRateYaw)`，允许慢速修正方向 |
| 攻击结束 | `ResetAttack -> RestoreLocomotionRotation`，恢复普通移动转向 |

接入后测试发现：

```text
镜头转向后，角色在攻击期间没有平滑转向镜头方向。
```

通过截图和 Unreal MCP 校对后，确认当前攻击 Montage 使用 Root Motion。Root Motion 播放期间，`Character Movement` 的旋转控制会受到限制，因此仅设置：

```text
Orient Rotation to Movement = false
Use Controller Desired Rotation = true
Rotation Rate = AttackStartTurnRateYaw
```

仍不足以稳定生效。

随后采用的修复方式：

| 位置 | 修改 |
|---|---|
| `BP_PlayerCharacter.SetAttackTurnEnabled` | 在原有攻击转向逻辑前新增 `Set Allow Physics Rotation During Anim Root Motion = true` |
| `BP_PlayerCharacter.RestoreLocomotionRotation` | 在原有恢复移动转向逻辑中新增 `Set Allow Physics Rotation During Anim Root Motion = false` |
| `BP_PlayerCharacter.ResetAttack` | 保持调用 `RestoreLocomotionRotation`，确保攻击结束后恢复普通移动转向 |

中文节点名：

```text
允许动画根运动期间物理旋转
朝移动方向旋转
使用控制器期望旋转
旋转速率
```

英文节点名：

```text
Allow Physics Rotation During Anim Root Motion
Orient Rotation to Movement
Use Controller Desired Rotation
Rotation Rate
```

测试结论：

```text
完成第 1、2 部分修复后，攻击期间角色已经可以随镜头方向转向。
由于当前表现已经满足需求，暂不继续加入 Timeline / RInterp To 起手强制转向，避免和 Root Motion 抢控制。
```
