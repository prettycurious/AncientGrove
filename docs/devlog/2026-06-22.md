## 一、Unreal MCP 初始状态排查

### 1. 目的：根据 Codex 终端截图判断 `unreal-mcp` 是否连接成功

用户提供的 Codex 终端截图中，`/mcp` 输出显示 `unreal-mcp` 配置已存在，但启动失败。

关键报错为：

```text
MCP client for `unreal-mcp` failed to start
MCP startup failed: handshaking with MCP server failed
HTTP request failed: http://127.0.0.1:8000/mcp
MCP startup incomplete (failed: unreal-mcp)
```

截图中还显示：

```text
unreal-mcp
Auth: Unsupported
Tools: (none)
```

当时判断结果：

- Codex 认识 `unreal-mcp` 这个 MCP 配置名；
- Codex 没有成功连接到 `http://127.0.0.1:8000/mcp`；
- `Tools: (none)` 表示没有成功发现可用工具；
- 因此当时 `unreal-mcp` 连接状态为失败。

### 2. 目的：记录初始失败时的可能原因

根据报错，初始失败重点集中在 Codex 无法访问本机 MCP HTTP 地址：

```text
http://127.0.0.1:8000/mcp
```

当时建议优先检查：

- Unreal Editor 是否已经打开 `AncientGrove` 项目；
- Unreal MCP 插件是否已启用；
- MCP 服务是否实际监听 `127.0.0.1:8000`；
- Codex MCP 配置中的端口是否和 UE 端实际端口一致；
- UE 输出日志中是否存在 MCP server started / listening 相关信息。

建议查看 UE 日志的位置为：

```text
窗口（Window）-> 开发者工具（Developer Tools）-> 输出日志（Output Log）
```

### 2. 目的：确认当前暴露的 MCP 工具范围

继续读取 `ToolsetRegistry.AgentSkillToolset` 的工具详情，当前暴露的工具为：

```text

GetSkills
CreateSkill
UpdateSkill
```

这些工具主要用于管理项目中的 `AgentSkill`：

- `GetSkills`：读取指定 AgentSkills 的详细信息；
- `CreateSkill`：创建新的 AgentSkill；
- `UpdateSkill`：更新现有 AgentSkill。

其中 `CreateSkill` 和 `UpdateSkill` 属于会修改项目内容的操作，后续只有在用户明确要求或授权后才能调用。

### 3. 目的：使用只读工具校对 UE MCP 是否能正常返回项目数据

调用只读工具 `ListSkills` 后，返回结果为：

```json
{"returnValue":{}}
```

校对结果：

- `ListSkills` 调用成功，说明 MCP 通道可以正常完成一次工具调用；
- 返回空字典表示当前项目没有列出任何 AgentSkill；
- 返回空字典不是连接失败，而是当前没有可列出的 AgentSkill 数据。

- `SetPercent` 的目标为 `PB_Health`；
