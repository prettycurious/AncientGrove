# 2026-06-23 开发日志

## 今日目标

今天主要把敌人血量显示从“敌人头顶血条”调整为“Boss 战 HUD 血条”，并对相关蓝图进行只读校对。

当前项目只有一个敌人，且敌人已经作为 Boss 设计，因此屏幕底部居中的 Boss 血条比头顶血条更合适：

- 更符合 Boss 战表现；
- 不会因为镜头、距离或朝向导致头顶血条难以阅读；
- 后续可以继续扩展 Boss 名称、阶段提示、特殊技能提示等内容；
- 当前 UI 统一放在 `WBP_PlayerHUD` 中，便于 v1.0 收尾。

本日蓝图资源修改主要由用户在 Unreal Editor 中手动完成；Codex 侧通过 Unreal MCP 做只读检查，并在最后更新文档。本文档记录当前已校对通过的结构、节点关系和后续测试清单。

### 2. 目的：确认 Boss 血条函数

已校对 `WBP_PlayerHUD` 中存在以下函数：

```text
ShowBossHealth
HideBossHealth
UpdateBossHealth
```

`ShowBossHealth` 当前逻辑：

```text
Function Entry
-> BossHealthBox.SetVisibility(Visible)
```

`HideBossHealth` 当前逻辑：

```text
Function Entry
-> BossHealthBox.SetVisibility(Hidden)
```

`UpdateBossHealth` 当前逻辑：

```text
Function Entry
-> Safe Divide(CurrentHP, MaxHP)
-> Clamp(Float), Min = 0, Max = 1
-> PB_BossHP.SetPercent
```

校对结果：

- `UpdateBossHealth` 已从普通除法改为 `Safe Divide`；
- `CurrentHP` 接入 `Safe Divide` 的 `A`；
- `MaxHP` 接入 `Safe Divide` 的 `B`；
- `Safe Divide` 输出进入 `Clamp(Float)`；
- `Clamp(Float)` 输出进入 `PB_BossHP.SetPercent`；
- 该结构可以避免 `MaxHP` 意外为 `0` 时出现除零问题。

## 三、BP_PlayerCharacter 校对

### 1. 目的：降低 Boss 对 HUD 的直接耦合

当前没有让 `BP_AncientGuardian` 直接访问：

```text
PlayerHUDRef
```

而是在 `BP_PlayerCharacter` 中新增转发函数：

```text
ShowBossHealthOnHUD
UpdateBossHealthOnHUD
HideBossHealthOnHUD
```

这样 Boss 只需要认识玩家角色 `PlayerRef`，由玩家角色负责把请求转发到 `WBP_PlayerHUD`。

### 2. 目的：确认玩家转发函数

`ShowBossHealthOnHUD` 当前逻辑：

```text
Function Entry
-> Is Valid? (PlayerHUDRef)
-> PlayerHUDRef.ShowBossHealth
```

`UpdateBossHealthOnHUD` 当前逻辑：

```text
Function Entry
-> Is Valid? (PlayerHUDRef)
-> PlayerHUDRef.UpdateBossHealth(BossCurrentHP, BossMaxHP)
```

`HideBossHealthOnHUD` 当前逻辑：

```text
Function Entry
-> Is Valid? (PlayerHUDRef)
-> PlayerHUDRef.HideBossHealth
```

校对结果：

- 三个转发函数都先判断 `PlayerHUDRef` 是否有效；
- `UpdateBossHealthOnHUD` 的输入参数为 `BossCurrentHP` 和 `BossMaxHP`；
- 之前误接玩家自身 `CurrentHP / MaxHP` 的风险已经排除；
- Boss 血条刷新使用 Boss 自己传入的血量。

## 八、补充记录：Boss 追踪分支修复

### 1. 目的：记录“第一次追踪后不再追踪”的原因

在 Boss 血条改为进入 `DetectRange` 后显示时，最初曾把 `AI MoveTo` 接在 Boss 血条首次显示链路后面：

```text
Set BossHealthShown = true
-> ShowBossHealthOnHUD
-> UpdateBossHealthOnHUD
-> AI MoveTo
```

这个结构会导致：

```text
BossHealthShown 第一次变为 true 后，后续不再进入该分支；
AI MoveTo 也就不再持续执行。
```

后续拆分为 `Sequence` 两条分支后，又发现一次接线风险：

```text
AI MoveTo 如果接在 CanAttack 分支的 False 上，也会导致敌人不追踪。
```

原因是：

```text
CanAttack = false 只代表攻击冷却中；
真正需要追踪的情况是 Distance > AttackRange。
```

### 2. 目的：记录最终正确接线

当前正确结构应为：

```text
Sequence Then 0：追踪和攻击
-> Branch: Distance <= DetectRange
   True:
   -> Branch: Distance <= AttackRange
      True:
      -> Branch: CanAttack
         True:
         -> Set CanAttack = false
         -> Apply Damage
         -> PrintString("Enemy Attack")
         -> Delay(AttackInterval)
         -> Set CanAttack = true

         False:
         -> 空着，等待攻击冷却

      False:
      -> AI MoveTo

Sequence Then 1：Boss 血条首次显示
-> Branch: Distance <= DetectRange AND NOT BossHealthShown AND NOT IsDead
   True:
   -> Set BossHealthShown = true
   -> ShowBossHealthOnHUD
   -> UpdateBossHealthOnHUD
```

关键结论：

- `AI MoveTo` 必须接在 `AttackRange` 判断的 `False` 分支；
- `AI MoveTo` 不应接在 `CanAttack` 判断的 `False` 分支；
- `CanAttack = false` 时表示 Boss 正在等待攻击冷却，这个分支可以空着；
- Boss 血条显示分支只负责显示和刷新 HUD，不负责追踪。

### 3. 目的：记录修复后的测试重点

后续播放测试时，需要重点确认：

| 场景 | 预期结果 |
|---|---|
| 玩家在 `DetectRange` 外 | Boss 不追踪 |
| 玩家进入 `DetectRange` 且在 `AttackRange` 外 | Boss 执行 `AI MoveTo` 持续追踪 |
| 玩家进入 `AttackRange` 且 `CanAttack = true` | Boss 攻击玩家 |
| 玩家进入 `AttackRange` 但 `CanAttack = false` | Boss 等待冷却，不重复攻击 |
| 攻击冷却结束 | `CanAttack` 重新变为 `true` |

## 九、补充记录：HUD 底部信息重叠修复

### 1. 目的：记录问题现象

在 Boss 血条改为屏幕底部中间显示后，发现原本位于底部中间的交互提示会和 Boss 信息重叠。

可能同时显示的内容包括：

```text
TXT_InteractPrompt
BossHealthBox
TXT_BossName
PB_BossHP
```

典型触发场景：

```text
玩家已经进入 Boss 战，Boss 血条显示；
玩家靠近宝箱，触发“按 E 打开宝箱”等交互提示；
交互提示和 Boss 血条位于相近的底部区域。
```

### 2. 目的：记录修复方式

当前修复原则：

```text
BossHealthBox 保持屏幕底部中间
TXT_InteractPrompt 上移到 BossHealthBox 上方
```

这样 Boss 血条作为战斗核心信息保持稳定，交互提示作为临时信息显示在其上方。

建议校对位置：

```text
TXT_InteractPrompt Position Y = -160 到 -180
```

具体位置可以按实际画面微调，核心要求是：

- 不遮挡 `TXT_BossName`；
- 不遮挡 `PB_BossHP`；
- 和 `BossHealthBox` 保持 30 到 50 像素左右的视觉间距；
- 移动到移动端或不同分辨率时仍应重新校对。

### 3. 目的：记录后续测试项

后续播放测试时，需要加入以下 UI 校对：

| 场景 | 预期结果 |
|---|---|
| Boss 血条显示时靠近宝箱 | `TXT_InteractPrompt` 显示在 Boss 血条上方 |
| Boss 血条显示时按 `E` 触发失败提示 | “请先击败遗迹守卫”等提示不遮挡 Boss 血条 |
| 不靠近宝箱按 `E` | “附近没有可交互对象”不遮挡 Boss 血条 |
| Boss 死亡后 | Boss 血条隐藏，交互提示仍正常显示 |
