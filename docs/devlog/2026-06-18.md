# 2026-06-18 开发日志

## 今日目标

排查玩家左键攻击时出现“附近没有可交互对象”的问题，并继续校对 v1.0 HUD 血量刷新链路。

## 排查结论

截图确认问题发生在 `BP_PlayerCharacter`：

1. `EnhancedInputAction IA_Attack` 是左键攻击输入；
2. 当前攻击输入最前面接了 `CurrentInteractChest -> Is Valid`；
3. 当 `CurrentInteractChest` 无效时，会调用 `ShowHUDPrompt("附近没有可交互对象")`；
4. 所以玩家点击左键攻击时，如果附近没有宝箱，就会误显示交互失败提示。

这不是宝箱检测范围本身的问题，而是攻击输入和交互输入的职责混在了一起。

## 修正方案

1. `IA_Attack` 只保留攻击逻辑：
   - 判断 `IsAttacking`；
   - 设置 `IsAttacking = true`；
   - 播放 `AM_PlayerAttack`；
   - 延迟后执行攻击 Trace；
   - 攻击结束后恢复 `IsAttacking = false`。
2. 将 `CurrentInteractChest -> Is Valid -> TryOpenChest / ShowHUDPrompt("附近没有可交互对象")` 移到 `IA_Interact` 输入事件中。
3. 检查 `IMC_Player`：
   - `IA_Attack` 绑定左键 `Left Mouse Button`；
   - `IA_Interact` 绑定 `E` 键。

## 第一轮校对结果

1. 当前截图中的节点名显示为 `EnhancedInputAction IA_Attack`，确认是攻击输入事件；
2. “附近没有可交互对象”来自 `ShowHUDPrompt`，且接在 `IA_Attack` 无交互对象分支后；
3. 下一步应在 UE 中按修正方案重接蓝图，再进行左键攻击和 E 键交互测试。

## 第二轮校对结果

根据重新接线后的截图，当前蓝图状态如下：

1. `IA_Attack` 已只保留攻击流程：
   - `Started -> Branch`；
   - `Branch` 条件为 `NOT IsAttacking`；
   - 成功后执行 `Set IsAttacking = true -> Play Anim Montage -> Delay -> Sphere Trace By Channel`。
2. `IA_Interact` 已独立负责交互流程：
   - `Started -> Is Valid(CurrentInteractChest)`；
   - 有效时调用 `TryOpenChest`；
   - 无效时调用 `ShowHUDPrompt("附近没有可交互对象")`。
3. `IMC_Player` 映射已校对：
   - `IA_Attack` 绑定鼠标左键 `Left Mouse Button`；
   - `IA_Interact` 绑定 `E` 键。

## 待测试项

1. 点击 编译（Compile）并确认没有蓝图报错；
2. 点击 保存（Save）或 文件（File）-> 保存全部（Save All）；
3. 播放（Play）后测试左键空挥不再显示“附近没有可交互对象”；
4. 不靠近宝箱按 `E` 时仍显示“附近没有可交互对象”；
5. 靠近宝箱按 `E` 时仍能调用 `TryOpenChest`。

## 第三轮问题

不靠近宝箱按 `E` 后，“附近没有可交互对象”会一直存在。

原因是 `IA_Interact` 的无效分支只调用了 `ShowHUDPrompt("附近没有可交互对象")`，但没有后续调用 `ClearHUDPrompt`。`ShowHUDPrompt` 负责显示提示，不会自动隐藏。

## 第三轮修正方案

只在 `IA_Interact` 的 `Is Not Valid` 分支后追加临时清理流程：

```text
Is Not Valid
-> ShowHUDPrompt("附近没有可交互对象")
-> Delay(1.5)
-> Is Valid(CurrentInteractChest)
-> Is Not Valid 时调用 ClearHUDPrompt
```

这样做的目的：

1. 不靠近宝箱按 `E` 时，提示显示约 1.5 秒后自动隐藏；
2. 如果提示显示期间玩家走进宝箱范围，`CurrentInteractChest` 会变为有效，不会把新的“按 E 打开宝箱”提示误清掉。

## 第三轮校对结果

根据最新截图，`IA_Interact` 接线已符合修正方案：

1. `Started -> Is Valid(CurrentInteractChest)`；
2. `Is Valid -> TryOpenChest`；
3. `Is Not Valid -> ShowHUDPrompt("附近没有可交互对象") -> Delay(1.5)`；
4. `Delay Completed -> Is Valid(CurrentInteractChest)`；
5. 第二个 `Is Valid` 的 `Is Not Valid -> ClearHUDPrompt`。

待进入播放（Play）后验证：不靠近宝箱按 `E` 时提示约 1.5 秒后自动隐藏；靠近宝箱期间不会被旧延迟误清掉。

## 第四轮问题

玩家受到敌人伤害后，`CurrentHP` 已经扣减，但 HUD 上的血条和血量文本没有刷新。

根据截图校对，`BP_PlayerCharacter` 的伤害流程中，`Set CurrentHP` 后已经调用 `RefreshHUDHealth`，因此问题重点不在扣血节点，而在 HUD 引用和 `UpdateHealth` 内部接线。

## 第四轮排查结果

根据后续截图和测试反馈，当前血量刷新链路已验证通过：

1. `BP_PlayerCharacter` 的 `BeginPlay` 已创建 `WBP_PlayerHUD`；
2. `创建控件（Create Widget）` 的返回值已保存到 `PlayerHUDRef`；
3. `PlayerHUDRef` 已用于 `添加到视口（Add to Viewport）`；
4. `RefreshHUDHealth` 已先判断 `PlayerHUDRef` 是否有效，再调用 `UpdateHealth`；
5. `WBP_PlayerHUD.UpdateHealth` 已通过 `PlayerCharacterRef` 读取 `CurrentHP` 和 `MaxHP`；
6. `CurrentHP / MaxHP` 经过 `Clamp(Float)` 后设置到 `PB_PlayerHP.Percent`；
7. `TXT_HPValue` 已通过格式化文本显示当前血量和最大血量；
8. 玩家扣血后 HUD 血条和血量文本已可以正常刷新。
