# v0.6 打斗精修实施记录

## 1. 文档目的

本文档用于独立记录 `AncientGrove` 的 v0.6 打斗精修实施过程。

v0.6 位于：

```text
v0.5 宝箱交互闭环
-> v0.6 打斗精修
-> v1.0 可玩 Demo
```

本阶段目标不是增加大量新系统，而是把现有“可以攻击”的战斗，精修为“有基础手感”的战斗。

当前重点：

1. 玩家 3 段轻攻击连击；
2. Animation Montage 分段播放；
3. Anim Notify 控制连击窗口和攻击命中时机；
4. Sphere Trace By Channel 做攻击命中检测；
5. 每段攻击只对同一敌人造成一次伤害；
6. 每段攻击可配置不同伤害；
7. Boss 受击后掉血、硬直、停止移动并击退；
8. Boss 血条继续正常刷新；
9. 攻击时角色可以在 Root Motion Montage 中平滑转向镜头方向；
10. 后续继续微调命中特效、攻击滑步和攻击期间移动手感。

## 4. AM_Player_Attack_Combo

### 4.1 目的：用一个 Montage 管理 3 段轻攻击

当前 Montage：

```text
AM_Player_Attack_Combo
```

包含 3 段攻击动画：

```text
MM_Attack_01
MM_Attack_02
MM_Attack_03
```

当前 Section：

```text
Attack_1
Attack_2
Attack_3
```

当前 Slot：

```text
DefaultSlot
```

当前混合设置：

```text
Blend In Time = 0.05
Blend Out Time = 0.10
Rate Scale = 1.0
```

校对结论：

- 3 段攻击均在同一个 Montage 内；
- `Play Anim Montage` 可以通过 `Start Section Name` 播放指定攻击段；
- 后续继续新增攻击动作时，优先保持 Section 命名清晰。

### 4.2 目的：记录 Montage Notify 约定

当前使用的 Notify：

| Notify | 作用 |
|---|---|
| `AttackHitStart` | 清空本段已命中敌人列表 |
| `AttackHitCheck` | 执行一次攻击命中检测 |
| `ComboWindowOpen` | 打开连击窗口，允许缓存下一段攻击 |
| `ComboWindowClose` | 关闭连击窗口，并根据缓存决定是否播放下一段 |
| `AttackEnd` | 最后一段攻击结束后重置攻击状态 |

当前推荐摆放规则：

| 攻击段 | Notify 建议 |
|---|---|
| `Attack_1` | `AttackHitStart`、`AttackHitCheck`、`ComboWindowOpen`、`ComboWindowClose` |
| `Attack_2` | `AttackHitStart`、`AttackHitCheck`、`ComboWindowOpen`、`ComboWindowClose` |
| `Attack_3` | `AttackHitStart`、`AttackHitCheck`、`AttackEnd` |

说明：

```text
Attack_1 和 Attack_2 不依赖 AttackEnd 重置。
如果没有缓存下一段，CloseComboWindow 会调用 ResetAttack。
Attack_3 是最后一段，结束时由 AttackEnd 重置。
```

## 6. BP_PlayerCharacter 连击系统

### 6.1 目的：记录玩家连击变量

当前玩家连击相关变量：

| 变量名 | 类型 | 当前用途 |
|---|---|---|
| `IsAttacking` | Boolean | 是否正在攻击 |
| `CanCombo` | Boolean | 当前是否允许输入下一段 |
| `HasQueuedCombo` | Boolean | 玩家是否已在连击窗口内缓存下一段 |
| `AttackIndex` | Integer | 当前攻击段数 |
| `MaxCombo` | Integer | 最大连击段数，当前为 3 |
| `HitActors` | Actor Array | 当前攻击段已经命中过的 Actor |
| `AttackTraceRadius` | Float | Sphere Trace 半径 |
| `AttackDamage` 或分段伤害变量 | Float | 攻击伤害配置 |

### 6.2 目的：记录 IA_Attack 输入逻辑

当前 `IA_Attack` 使用：

```text
EnhancedInputAction IA_Attack
Started
```

主逻辑：

```text
IA_Attack Started
-> Branch: NOT IsAttacking
   True:
      Set IsAttacking = true
      Set CanCombo = false
      Set HasQueuedCombo = false
      Set AttackIndex = 1
      Play Anim Montage
         Montage = AM_Player_Attack_Combo
         Start Section Name = Attack_1

   False:
      Branch: CanCombo
         True:
            Set HasQueuedCombo = true
         False:
            空置
```

校对结论：

- 不攻击时，左键会从 `Attack_1` 开始；
- 已在攻击中但还没进入连击窗口时，左键不会乱触发；
- 已在攻击中且 `CanCombo = true` 时，左键会缓存下一段；
- 当前不再使用早期固定 Delay 后直接 Trace 的旧攻击链路。

### 6.3 目的：记录 ResetAttack

`ResetAttack` 当前职责：

```text
Set CanCombo = false
Set HasQueuedCombo = false
Set AttackIndex = 0
Set IsAttacking = false
```

作用：

- 攻击结束后恢复可再次攻击状态；
- 未缓存下一段时，也可以在 `CloseComboWindow` 中主动调用；
- 防止只打第一段后玩家无法再次攻击。

### 6.4 目的：记录 OpenComboWindow

`OpenComboWindow` 当前职责：

```text
Set CanCombo = true
```

作用：

```text
只有连击窗口打开后，下一次左键才会被缓存为下一段攻击。
```

### 6.5 目的：记录 CloseComboWindow

`CloseComboWindow` 当前职责：

```text
Set CanCombo = false
-> Branch: HasQueuedCombo
   False:
      ResetAttack
   True:
      根据 AttackIndex 播放下一段
```

推荐结构：

```text
Branch: HasQueuedCombo
False:
   ResetAttack

True:
   Branch: AttackIndex == 1
      True:
         Set HasQueuedCombo = false
         Set AttackIndex = 2
         Play Anim Montage
            Montage = AM_Player_Attack_Combo
            Start Section Name = Attack_2

      False:
         Branch: AttackIndex == 2
            True:
               Set HasQueuedCombo = false
               Set AttackIndex = 3
               Play Anim Montage
                  Montage = AM_Player_Attack_Combo
                  Start Section Name = Attack_3

            False:
               ResetAttack
```

重要记录：

```text
曾测试 Montage Jump to Section / Montage Set Next Section。
当时出现 PrintString 已打印但 Attack_2 未播放的问题。
当前稳定方案是重新调用 Play Anim Montage，并指定 Start Section Name。
```

### 6.6 目的：记录同名节点区分

蓝图里存在多个同名节点，需要按上下文区分：

| 同名节点 | 正确位置 |
|---|---|
| `Branch` | `IA_Attack` 下的第一个 `Branch` 判断 `NOT IsAttacking` |
| `Branch` | `IA_Attack` 的攻击中分支判断 `CanCombo` |
| `Play Anim Montage` | 开始攻击时播放 `Attack_1` |
| `Play Anim Montage` | `CloseComboWindow` 中播放 `Attack_2` 或 `Attack_3` |
| `ResetAttack` | 玩家函数，不是 Anim Notify 名字 |

## 7. BP_PlayerCharacter 命中检测

### 7.1 目的：每段攻击开始时清空已命中列表

`AttackHitStart` 当前职责：

```text
Clear HitActors
```

作用：

```text
保证每一段攻击可以重新命中敌人，但同一段攻击不会重复扣血。
```

### 7.2 目的：用 Sphere Trace 检测前方敌人

`AttackHitCheck` 当前职责：

```text
Get Actor Location
Get Actor Forward Vector
Start = Actor Location
End = Actor Location + Forward Vector * 攻击距离
Sphere Trace By Channel
```

中文节点名：

```text
按通道球体追踪
```

英文节点名：

```text
Sphere Trace By Channel
```

命中后处理：

```text
Break Hit Result
-> Hit Actor
-> 判断 Hit Actor 是否已经在 HitActors 中
-> 未命中过：
   Add HitActors
   Apply Damage
```

校对结论：

- 攻击检测由玩家角色发起；
- 命中敌人后调用 UE 的 `Apply Damage`；
- `Damage Causer` 使用玩家自身；
- 同一段攻击不会重复伤害同一个 Actor。

### 7.3 目的：记录分段伤害

当前已验证分段伤害：

```text
Attack_1 = 20
Attack_2 = 25
Attack_3 = 35
```

说明：

```text
分段伤害用于让连击递进有收益。
后续可以改成数组、DataTable 或函数统一管理。
当前蓝图方案先保持简单。
```

## 8. BP_AncientGuardian 受击、硬直和击退

### 8.1 目的：记录 Boss 受击入口

当前 Boss 受击使用：

```text
Event AnyDamage
```

主流程：

```text
Event AnyDamage
-> Branch: NOT IsDead
-> Set CurrentHP = CurrentHP - Damage
-> UpdateBossHealthOnHUD(CurrentHP, MaxHP)
-> PrintString("Enemy Hit")
-> Branch: CurrentHP <= 0
```

死亡分支：

```text
Set IsDead = true
-> Delay(1.0)
-> HideBossHealthOnHUD
-> Set BossHealthShown = false
-> PrintString("Enemy Dead")
-> DestroyActor
```

非死亡分支：

```text
Set IsHitStunned = true
-> Stop Movement Immediately
-> Launch Character
-> Retriggerable Delay(HitStunTime)
-> Set IsHitStunned = false
```

### 8.2 目的：记录 Boss 硬直变量

当前 Boss 硬直与击退变量：

| 变量名 | 类型 | 当前默认值 | 作用 |
|---|---|---:|---|
| `IsHitStunned` | Boolean | false | 是否处于受击硬直 |
| `HitStunTime` | Float | 0.25 | 硬直持续时间 |
| `HitKnockbackStrength` | Float | 450 | 水平击退力度 |
| `HitKnockbackUpStrength` | Float | 80 | 向上击退力度 |

Unreal MCP 只读检查结果：

```text
HitStunTime = 0.25
HitKnockbackStrength = 450
HitKnockbackUpStrength = 80
IsHitStunned = false
IsDead = false
```

### 8.3 目的：记录硬直对 AI 的阻断

`Event Tick` 中的 AI 逻辑需要先检查：

```text
IsHitStunned
```

推荐结构：

```text
Branch: IsHitStunned
True:
   空置，不追击，不攻击
False:
   继续原本 DetectRange / AttackRange / CanAttack 逻辑
```

校对结论：

- Boss 受击硬直期间不会继续追击；
- Boss 受击硬直期间不会继续攻击；
- 硬直结束后恢复原本行为。

### 8.4 目的：记录停止移动

当前使用节点：

```text
Stop Movement Immediately
```

中文节点名：

```text
立即停止移动
```

英文节点名：

```text
Stop Movement Immediately
```

Target：

```text
Character Movement
```

作用：

```text
清除 Boss 当前移动速度，避免受击后仍沿之前 AI MoveTo 指令滑动。
```

### 8.5 目的：记录击退计算

当前击退方向：

```text
BossLocation - PlayerLocation
```

节点结构：

```text
Get Actor Location(self)
Get Actor Location(PlayerRef)
Vector - Vector
Normalize
Vector * Float(HitKnockbackStrength)
```

向上分量：

```text
Make Vector
X = 0
Y = 0
Z = HitKnockbackUpStrength
```

最终速度：

```text
(水平击退向量) + (向上击退向量)
-> Launch Character.Launch Velocity
```

`Launch Character` 设置：

```text
Target = self
XYOverride = true
ZOverride = true
```

校对结论：

- Boss 会远离玩家方向后退；
- 变量已接入，不再依赖手填节点常量；
- 当前击退手感测试通过。

### 11.1 目的：解释为什么正式测试时停顿不明显

正式链路是：

```text

-> Launch Character
```

因此视觉上最明显的是：

```text
Boss 后退
```

而不是：

```text
Boss 原地冻结
```

结论：

```text
硬直逻辑生效。
正式表现中，硬直主要体现为后退期间不追击、不攻击。
```

### 11.9 目的：记录攻击转向接入与 Root Motion 修复

2026-06-29 已完成攻击期间角色转向优化。实际实施顺序为：

```text
先增加攻击转向系统；
再修复攻击 Montage Root Motion 下转向没有生效的问题。
```

第一步已经接入的攻击转向结构：

| 阶段 | 处理方式 |
|---|---|
| 攻击起手 | 调用 `SetAttackTurnEnabled(true, AttackStartTurnRateYaw)`，允许角色朝镜头方向平滑转向 |
| 命中阶段 | `AnimNotify_AttackTurnLock` 调用 `SetAttackTurnEnabled(false, 0)`，锁定攻击朝向 |
| 收招阶段 | `AnimNotify_AttackTurnRecover` 调用 `SetAttackTurnEnabled(true, AttackRecoverTurnRateYaw)`，允许较慢修正方向 |
| 攻击结束 | `ResetAttack -> RestoreLocomotionRotation`，恢复正常移动转向 |

第二步发现的问题表现为：

```text
镜头转向后，角色在攻击 Montage 播放期间没有平滑转向镜头方向。
```

排查结论：

```text
攻击 Montage 当前使用 Root Motion。
Root Motion 播放期间，Character Movement 的旋转控制会被限制。
Unreal MCP 校对时确认 Character Movement 的 bAllowPhysicsRotationDuringAnimRootMotion 原本为 false。
```

最终采用的方案：

```text
在已经接入攻击转向的基础上，允许 Root Motion 期间的物理旋转；
在攻击结束恢复移动时，再恢复为关闭状态。
```

涉及函数：

| 函数 | 修改内容 |
|---|---|
| `SetAttackTurnEnabled` | 原本负责攻击转向开关和 `Rotation Rate` 设置；修复时新增 `Set Allow Physics Rotation During Anim Root Motion = true` |
| `RestoreLocomotionRotation` | 原本负责恢复普通移动转向；修复时新增 `Set Allow Physics Rotation During Anim Root Motion = false` |
| `ResetAttack` | 已调用 `RestoreLocomotionRotation`，确保攻击结束后恢复普通移动旋转规则 |

中文节点名：

```text
设置攻击转向启用（SetAttackTurnEnabled）
恢复移动旋转（RestoreLocomotionRotation）
允许动画根运动期间物理旋转（Allow Physics Rotation During Anim Root Motion）
朝移动方向旋转（Orient Rotation to Movement）
使用控制器期望旋转（Use Controller Desired Rotation）
旋转速率（Rotation Rate）
```

英文节点名：

```text
SetAttackTurnEnabled
RestoreLocomotionRotation
Set Allow Physics Rotation During Anim Root Motion
Set Orient Rotation to Movement
Set Use Controller Desired Rotation
Set Rotation Rate
```

未采用的增强方案：

```text
暂不加入 Timeline / RInterp To 起手强制转向。
原因：开启 Root Motion 期间物理旋转后，当前测试已经可以转向；继续增加 Timeline 会提高和 Root Motion 抢控制的风险。
```

测试结论：
