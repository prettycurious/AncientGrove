# v1.0 Boss 战血条实施步骤

本文记录当前敌人血条方案。由于当前只有一个敌人，并且敌人已经按 Boss 设计处理，所以当前方案不是敌人头顶血条，而是屏幕底部 Boss 战 HUD 血条。

菜单名称同时写中文和英文，方便在中文 Unreal Editor 界面中查找。

## 一、当前方案结论

### 1. 目的：明确当前不再使用头顶血条

已撤销的旧方案：

```text
WBP_EnemyHealthBar
BP_AncientGuardian.EnemyHealthBar
BP_AncientGuardian.UpdateEnemyHealthBar
```

当前方案：

```text
WBP_PlayerHUD 内显示 Boss 血条
BP_PlayerCharacter 负责转发 HUD 调用
BP_AncientGuardian 在玩家进入 DetectRange 后触发 Boss 血条显示
```

当前设计取舍：

- 开局 Boss 血条隐藏；
- 玩家第一次进入 `DetectRange` 后显示 Boss 血条；
- Boss 受伤后刷新 Boss 血条；
- Boss 死亡后隐藏 Boss 血条；
- 玩家离开追击范围后，Boss 血条当前继续显示，表示已经进入 Boss 战。

## 二、WBP_PlayerHUD 校对

### 1. 目的：打开 Boss 血条所在 HUD

操作路径：

```text
内容浏览器（Content Browser）-> Content -> AncientGrove -> UI
```

打开：

```text
WBP_PlayerHUD
```

切换到：

```text
设计器（Designer）
```

英文名称：

```text
Designer
```

### 2. 目的：确认 Boss 血条控件

在层级面板中确认存在以下控件：

```text
BossHealthBox
TXT_BossName
SizeBox_BossHP
PB_BossHP
```

控件职责：

| 控件 | 类型 | 作用 |
|---|---|---|
| `BossHealthBox` | 垂直框（Vertical Box） | Boss 血条整体容器 |
| `TXT_BossName` | 文本块（Text Block） | 显示 Boss 名称 |
| `SizeBox_BossHP` | 尺寸框（Size Box） | 固定血条高度 |
| `PB_BossHP` | 进度条（Progress Bar） | 显示 Boss 当前血量比例 |

推荐位置：

```text
屏幕底部居中，略高于底边
```

当前 `BossHealthBox` 建议初始可见性：

```text
隐藏（Hidden）
```

英文属性：

```text
Visibility = Hidden
```

### 3. 目的：确认 Boss 血条函数

切换到：

```text
图表（Graph）
```

英文名称：

```text
Graph
```

在：

```text
我的蓝图（My Blueprint）-> 函数（Functions）
```

英文路径：

```text
My Blueprint -> Functions
```

确认存在以下函数：

```text
ShowBossHealth
HideBossHealth
UpdateBossHealth
```

### 4. 目的：校对 ShowBossHealth

打开函数：

```text
ShowBossHealth
```

应看到逻辑：

```text
Function Entry
-> BossHealthBox.SetVisibility(Visible)
```

注意：

- 这里设置的是 `BossHealthBox`；
- 不是 `PB_BossHP`；
- 可见性使用 可见（Visible）/ `Visible`。

### 5. 目的：校对 HideBossHealth

打开函数：

```text
HideBossHealth
```

应看到逻辑：

```text
Function Entry
-> BossHealthBox.SetVisibility(Hidden)
```

注意：

- 隐藏目标仍然是 `BossHealthBox`；
- 可见性使用 隐藏（Hidden）/ `Hidden`。

### 6. 目的：校对 UpdateBossHealth

打开函数：

```text
UpdateBossHealth
```

确认输入参数：

| 参数 | 类型 | 说明 |
|---|---|---|
| `CurrentHP` | Float | Boss 当前血量 |
| `MaxHP` | Float | Boss 最大血量 |

应看到逻辑：

```text
Function Entry
-> Safe Divide(CurrentHP, MaxHP)
-> Clamp(Float), Min = 0, Max = 1
-> PB_BossHP.SetPercent
```

注意：

- 这里使用的是 安全除法（Safe Divide），不是普通 `/` 除法；
- `CurrentHP` 接到 `Safe Divide` 的 `A`；
- `MaxHP` 接到 `Safe Divide` 的 `B`；
- `Clamp(Float)` 的 `Min` 是 `0.0`，`Max` 是 `1.0`；
- 最后调用的是 `PB_BossHP` 的 设置百分比（Set Percent）/ `Set Percent`。

## 三、BP_PlayerCharacter 校对

### 1. 目的：打开玩家角色蓝图

操作路径：

```text
内容浏览器（Content Browser）-> Content -> AncientGrove -> Blueprints -> Character
```

打开：

```text
BP_PlayerCharacter
```

### 2. 目的：确认 Boss HUD 转发函数

在：

```text
我的蓝图（My Blueprint）-> 函数（Functions）
```

英文路径：

```text
My Blueprint -> Functions
```

确认存在以下函数：

```text
ShowBossHealthOnHUD
UpdateBossHealthOnHUD
HideBossHealthOnHUD
```

### 3. 目的：校对 ShowBossHealthOnHUD

打开函数：

```text
ShowBossHealthOnHUD
```

应看到逻辑：

```text
Function Entry
-> Is Valid? (PlayerHUDRef)
-> PlayerHUDRef.ShowBossHealth
```

注意：

- 这里的 有效性判断（Is Valid?）要使用带执行引脚的节点；
- `PlayerHUDRef` 是玩家角色保存的 HUD 引用；
- Boss 不应该直接访问 `PlayerHUDRef`。

### 4. 目的：校对 UpdateBossHealthOnHUD

打开函数：

```text
UpdateBossHealthOnHUD
```

确认输入参数：

| 参数 | 类型 | 说明 |
|---|---|---|
| `BossCurrentHP` | Float | 从 Boss 传入的当前血量 |
| `BossMaxHP` | Float | 从 Boss 传入的最大血量 |

应看到逻辑：

```text
Function Entry
-> Is Valid? (PlayerHUDRef)
-> PlayerHUDRef.UpdateBossHealth(BossCurrentHP, BossMaxHP)
```

注意：

- `BossCurrentHP` 要连接到 `UpdateBossHealth` 的 `CurrentHP` 输入；
- `BossMaxHP` 要连接到 `UpdateBossHealth` 的 `MaxHP` 输入；
- 不要误接玩家自己的 `CurrentHP` 和 `MaxHP`。

### 5. 目的：校对 HideBossHealthOnHUD

打开函数：

```text
HideBossHealthOnHUD
```

应看到逻辑：

```text
Function Entry
-> Is Valid? (PlayerHUDRef)
-> PlayerHUDRef.HideBossHealth
```

## 四、BP_AncientGuardian 校对

### 1. 目的：打开 Boss 蓝图

操作路径：

```text
内容浏览器（Content Browser）-> Content -> AncientGrove -> Blueprints -> Enemy
```

打开：

```text
BP_AncientGuardian
```

### 2. 目的：确认 Boss 血条状态变量

在变量列表中确认存在：

```text
BossHealthShown
```

类型：

```text
布尔（Boolean）
```

默认值：

```text
false
```

作用：

```text
防止 Boss 血条在 Event Tick 中每帧重复调用显示逻辑。
```

### 3. 目的：校对进入检测范围后显示 Boss 血条

在：

```text
事件图表（Event Graph）
```

英文名称：

```text
Event Graph
```

找到距离检测和追击逻辑附近，应看到首次显示 Boss 血条的条件：

```text
GetDistanceTo(PlayerRef) <= DetectRange
AND NOT BossHealthShown
AND NOT IsDead
```

通过后执行：

```text
Set BossHealthShown = true
-> PlayerRef.ShowBossHealthOnHUD
-> PlayerRef.UpdateBossHealthOnHUD(CurrentHP, MaxHP)
```

注意：

- `ShowBossHealthOnHUD` 是 `BP_PlayerCharacter` 上的函数；
- 这里的目标对象应该是 `PlayerRef`；
- `UpdateBossHealthOnHUD` 的 `BossCurrentHP` 接 Boss 自己的 `CurrentHP`；
- `UpdateBossHealthOnHUD` 的 `BossMaxHP` 接 Boss 自己的 `MaxHP`。

### 4. 目的：校对追踪和攻击分支

Boss 血条显示分支和追踪分支必须拆开。Boss 血条显示分支只负责首次显示 HUD，不负责执行 `AI MoveTo`。

正确结构：

```text
Sequence Then 0：追踪和攻击
-> Branch: Distance <= DetectRange
   True:
   -> Branch: Distance <= AttackRange
      True:
      -> Branch: CanAttack
         True:
         -> Set CanAttack = false
         -> Apply Damage
         -> PrintString("Enemy Attack")
         -> Delay(AttackInterval)
         -> Set CanAttack = true

         False:
         -> 空着，等待攻击冷却

      False:
      -> AI MoveTo

Sequence Then 1：Boss 血条首次显示
-> Branch: Distance <= DetectRange AND NOT BossHealthShown AND NOT IsDead
   True:
   -> Set BossHealthShown = true
   -> ShowBossHealthOnHUD
   -> UpdateBossHealthOnHUD
```

注意：

- `AI MoveTo` 要接在 `AttackRange` 判断的 `False` 分支；
- `AI MoveTo` 不要接在 `CanAttack` 判断的 `False` 分支；
- `CanAttack = false` 只表示 Boss 正在攻击冷却中，这时不需要重新追踪；
- Boss 血条的 `UpdateBossHealthOnHUD` 后面不要接 `AI MoveTo`。

`AI MoveTo` 参数应保持：

```text
Pawn = Self
Target Actor = PlayerRef
Acceptance Radius = 80.0
Stop on Overlap = true
```

### 5. 目的：校对 Boss 受伤后刷新血条

在 `BP_AncientGuardian` 的 `AnyDamage` 逻辑中，找到扣血后的执行链。

应看到类似逻辑：

```text
AnyDamage
-> Branch
-> Set CurrentHP
-> PlayerRef.UpdateBossHealthOnHUD(CurrentHP, MaxHP)
-> PrintString("Enemy Hit")
-> 死亡判断
```

注意：

- 刷新血条要放在 `Set CurrentHP` 之后；
- 这样 HUD 读到的是扣血后的新血量。

### 6. 目的：校对 Boss 死亡后隐藏血条

在死亡分支中，应看到类似逻辑：

```text
Set IsDead = true
-> Delay
-> PlayerRef.HideBossHealthOnHUD
-> Set BossHealthShown = false
-> PrintString("Enemy Dead")
```

注意：

- `HideBossHealthOnHUD` 仍然通过 `PlayerRef` 调用；
- 当前 `Set BossHealthShown = false` 放在死亡流程中是可以接受的；
- 如果以后要做 Boss 重生或多阶段 Boss，可以再单独整理状态机。


点击：

```text

```

然后点击：

```text
保存（Save）
```

或使用：

```text
文件（File）-> 保存全部（Save All）
```

英文路径：

```text
File -> Save All
```

### 2. 目的：编译并保存玩家和 Boss 蓝图

依次打开：

```text
BP_PlayerCharacter
BP_AncientGuardian
```

每个蓝图都点击：

```text
编译（Compile）-> 保存（Save）
```

英文按钮：

```text
Compile -> Save
```

### 3. 目的：测试开局隐藏

点击：

```text
播放（Play）
```

英文按钮：

```text
Play
```

开局站在远离 Boss 的位置，确认：

```text
Boss 血条不显示
```

### 4. 目的：测试进入 Boss 战后显示

控制玩家靠近 Boss，进入 `DetectRange`。

预期结果：

```text
Boss 血条在屏幕底部居中显示
Boss 名称显示为“遗迹守卫”
Boss 血条为满血
```

### 5. 目的：测试受伤刷新

用鼠标左键攻击 Boss。

英文输入：

```text
Left Mouse Button
```

预期结果：

```text
Boss 血条随 CurrentHP 下降
```

### 6. 目的：测试持续追踪

进入 `DetectRange` 后，和 Boss 保持一段距离，让玩家处于：

```text
Distance <= DetectRange
并且
Distance > AttackRange
```

预期结果：

```text
Boss 会持续执行 AI MoveTo 追踪玩家
```

如果 Boss 不追踪，优先检查：

```text
AI MoveTo 是否接在 AttackRange 分支的 False 上
```

### 7. 目的：测试攻击冷却

靠近 Boss，进入 `AttackRange`。

预期结果：

```text
Boss 会攻击玩家
攻击后等待 AttackInterval
冷却结束后再次攻击
```

如果 Boss 只攻击一次，优先检查：

```text
PrintString("Enemy Attack")
-> Delay(AttackInterval)
-> Set CanAttack = true
```

### 8. 目的：测试死亡隐藏

持续攻击 Boss 直到死亡。

预期结果：

```text
Boss 死亡后 Boss 血条隐藏
```

### 9. 目的：测试当前设计选择

进入 Boss 战后再离开追击范围。

当前预期结果：

```text
Boss 血条继续显示
```

说明：

```text
这表示玩家已经触发 Boss 战。后续如果要做“离开区域后隐藏 Boss 血条”，需要额外增加离开范围判断。
```

## 六、当前校对结论

当前 Boss 血条方案已经按以下结构校对通过：

```text
WBP_PlayerHUD
  ShowBossHealth
  HideBossHealth
  UpdateBossHealth

BP_PlayerCharacter
  ShowBossHealthOnHUD
  UpdateBossHealthOnHUD
  HideBossHealthOnHUD

BP_AncientGuardian
  BossHealthShown
  DetectRange 首次触发显示
  AttackRange 外 AI MoveTo 追踪
  AttackRange 内 CanAttack 控制攻击
  AnyDamage 后刷新
  死亡后隐藏
```

下一步重点不是重新接线，而是在播放（Play）模式中完整测试：

- 开局隐藏；
- 进入检测范围后显示；
- 在检测范围内持续追踪；
- 进入攻击范围后按冷却攻击；
- 受伤后刷新；
- 死亡后隐藏；
- 击败 Boss 后打开宝箱并显示通关提示。
