# v1.0 HUD 实施步骤

本文档记录 `WBP_PlayerHUD` 的创建和接线步骤。菜单名称同时写中文和英文，方便在中文 UE 界面中查找。

## 目标

本阶段只做第一版 Demo 必需 UI：

1. 玩家血条；
2. 宝箱交互提示；
3. 通关提示；
4. 玩家死亡失败提示。

完成后，原来用于验证的 `Print String` 可以暂时保留作为调试输出，但玩家实际看到的提示应来自 HUD。

## 当前验证状态

更新时间：2026-06-18

已验证内容：

1. `BP_PlayerCharacter` 的 `BeginPlay` 已创建 `WBP_PlayerHUD`，并保存到 `PlayerHUDRef` 后添加到视口；
2. `BP_PlayerCharacter` 的 `RefreshHUDHealth` 已通过 `PlayerHUDRef` 调用 `WBP_PlayerHUD.UpdateHealth`；
3. `WBP_PlayerHUD` 的 `PlayerCharacterRef` 已能正确引用当前玩家；
4. `WBP_PlayerHUD.UpdateHealth` 已能根据 `CurrentHP / MaxHP` 刷新 `PB_PlayerHP`；
5. `TXT_HPValue` 已能在玩家扣血后同步刷新血量文本。

仍需继续验证：

1. 交互提示 `TXT_InteractPrompt` 的显示和隐藏；
2. 通关提示 `TXT_ResultMessage`；
3. 玩家死亡提示“你已倒下”；
4. 从出生到战斗、开宝箱、通关或死亡的完整流程。

## 需要新增的资产

| 路径 | 资产名 | 类型 | 说明 |
|---|---|---|---|
| `Content/AncientGrove/UI/` | `WBP_PlayerHUD` | 控件蓝图（Widget Blueprint） | 主 HUD |

创建方式：

1. 在内容浏览器（Content Browser）中进入 `Content/AncientGrove/UI/`；
2. 右键（Right Click）空白处；
3. 选择 用户界面（User Interface）-> 控件蓝图（Widget Blueprint）；
4. 父类选择 用户控件（User Widget）；
5. 命名为 `WBP_PlayerHUD`。

## WBP_PlayerHUD 设计器

打开 `WBP_PlayerHUD` 后，在设计器（Designer）中添加以下控件。

| 控件名 | 类型 | 锚点 | 建议位置/尺寸 | 初始文本或状态 |
|---|---|---|---|---|
| `PB_PlayerHP` | 进度条（Progress Bar） | 左上（Top Left） | X 40, Y 40, W 320, H 24 | Percent = 1.0 |
| `TXT_HPValue` | 文本块（Text Block） | 左上（Top Left） | X 40, Y 68, W 320, H 24 | `HP 100 / 100` |
| `TXT_InteractPrompt` | 文本块（Text Block） | 下中（Bottom Center） | X -250, Y -120, W 500, H 50 | 文本留空，可见性（Visibility）= 隐藏（Hidden） |
| `TXT_ResultMessage` | 文本块（Text Block） | 居中（Center） | X -300, Y -80, W 600, H 120 | 文本留空，可见性（Visibility）= 隐藏（Hidden） |

建议样式：

1. `PB_PlayerHP` 使用红色或绿色填充，背景使用深灰色；
2. `TXT_InteractPrompt` 字号 28，水平居中；
3. `TXT_ResultMessage` 字号 56，水平居中；
4. 所有文字颜色先用白色，后续美化时再统一调整。

## WBP_PlayerHUD 变量和函数

在图表（Graph）中新增变量：

| 变量名 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `PlayerCharacterRef` | `BP_PlayerCharacter` Object Reference | None | 当前玩家引用 |

新增函数：

| 函数名 | 输入 | 作用 |
|---|---|---|
| `UpdateHealth` | 无 | 从 `PlayerCharacterRef` 读取 `CurrentHP / MaxHP`，刷新血条和血量文本 |
| `SetInteractPrompt` | `Message` Text | 设置 `TXT_InteractPrompt` 文本并显示 |
| `ClearInteractPrompt` | 无 | 隐藏 `TXT_InteractPrompt` |
| `ShowResultMessage` | `Message` Text | 设置 `TXT_ResultMessage` 文本并显示 |

### 事件构造（Event Construct）接线

在事件构造（Event Construct）中：

1. `Get Owning Player Pawn`；
2. `Cast To BP_PlayerCharacter`；
3. 成功后设置 `PlayerCharacterRef`；
4. 调用 `UpdateHealth`；
5. 调用 `ClearInteractPrompt`；
6. 将 `TXT_ResultMessage` 的可见性（Visibility）设置为 隐藏（Hidden）。

### UpdateHealth 接线

`UpdateHealth` 中：

1. 判断 `PlayerCharacterRef` 是否有效；
2. 读取 `CurrentHP` 和 `MaxHP`；
3. 计算 `CurrentHP / MaxHP`；
4. 使用 `限制浮点数（Clamp (Float)）` 限制到 `0.0` 到 `1.0`；
5. 调用 `PB_PlayerHP -> 设置百分比（Set Percent）`；
6. 设置 `TXT_HPValue` 文本为 `HP 当前值 / 最大值`。

## BP_PlayerCharacter 接线

在 `BP_PlayerCharacter` 中新增变量：

| 变量名 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `PlayerHUDRef` | `WBP_PlayerHUD` Object Reference | None | 当前 HUD 引用 |

### 事件开始运行（Event BeginPlay）创建 HUD

在事件开始运行（Event BeginPlay）中追加以下节点：

1. `创建控件（Create Widget）`，类（Class）选择 `WBP_PlayerHUD`；
2. 返回值（Return Value）提升为变量（Promote to Variable），变量名为 `PlayerHUDRef`；
3. 从 `PlayerHUDRef` 调用 `添加到视口（Add to Viewport）`。

如果 `BP_PlayerCharacter` 的事件开始运行（Event BeginPlay）已经有输入映射相关逻辑，HUD 创建节点接在原有事件开始运行（Event BeginPlay）执行线后面即可。

### 新增玩家转发事件

在 `BP_PlayerCharacter` 中新增 4 个自定义事件（Custom Event）：

| 事件名 | 输入 | 作用 |
|---|---|---|
| `ShowHUDPrompt` | `Message` Text | 如果 `PlayerHUDRef` 有效，调用 `SetInteractPrompt` |
| `ClearHUDPrompt` | 无 | 如果 `PlayerHUDRef` 有效，调用 `ClearInteractPrompt` |
| `ShowHUDResult` | `Message` Text | 如果 `PlayerHUDRef` 有效，调用 `ShowResultMessage` |
| `RefreshHUDHealth` | 无 | 如果 `PlayerHUDRef` 有效，调用 `UpdateHealth` |

### 血量变化后刷新 HUD

在玩家受到伤害、`CurrentHP` 被修改之后，调用 `RefreshHUDHealth`。此链路已在 2026-06-18 验证通过，玩家扣血后 HUD 血条和血量文本可以刷新。

在 `CurrentHP <= 0` 且设置 `IsDead = true` 的分支中，追加：

```text
ShowHUDResult("你已倒下")
```

## BP_TreasureChest 接线

在 `BP_TreasureChest` 中，将原来的调试提示同步转发到玩家 HUD。

### 玩家进入宝箱范围

在 `InteractionBox` 的 BeginOverlap 逻辑中，成功 Cast 到 `BP_PlayerCharacter` 并设置 `CurrentInteractChest` 后，调用：

```text
ShowHUDPrompt("按 E 打开宝箱")
```

### 玩家离开宝箱范围

在 `InteractionBox` 的 EndOverlap 逻辑中，清空 `CurrentInteractChest` 后，调用：

```text
ClearHUDPrompt()
```

### 宝箱打开判断

在 `TryOpenChest` 中：

1. 如果 `IsOpened = true`，调用 `ShowHUDPrompt("宝箱已经打开")`；
2. 如果仍有敌人存活，调用 `ShowHUDPrompt("请先击败遗迹守卫")`；
3. 如果满足开启条件并成功打开，调用 `ShowHUDResult("试炼完成")`。

## BP_PlayerCharacter 交互键接线

在 `IA_Interact` 输入事件中：

1. 如果 `CurrentInteractChest` 有效，继续调用宝箱 `TryOpenChest`；
2. 如果 `CurrentInteractChest` 无效，先显示临时提示，再延迟清理。

```text
EnhancedInputAction IA_Interact
-> Started
-> Is Valid? (CurrentInteractChest)

Is Valid
-> TryOpenChest

Is Not Valid
-> ShowHUDPrompt("附近没有可交互对象")
-> Delay(1.5)
-> Is Valid? (CurrentInteractChest)

第二个 Is Valid? 的 Is Not Valid
-> ClearHUDPrompt()
```

注意：

1. 这里使用的是带执行引脚的 有效性判断（Is Valid?）节点，不是只返回 Boolean 的纯节点；
2. 延迟（Delay）的 Duration 设置为 `1.5`；
3. 延迟结束后需要再次判断 `CurrentInteractChest`。如果玩家已经走进宝箱范围，`CurrentInteractChest` 会变为有效，此时不要调用 `ClearHUDPrompt`，避免把新的“按 E 打开宝箱”提示误清掉。

## 测试清单

| 场景 | 预期结果 |
|---|---|
| 开始游戏 | 左上角显示满血血条 |
| 玩家被敌人攻击 | 血条下降，血量文本更新 |
| 不靠近宝箱按 `E` | HUD 显示“附近没有可交互对象”，约 1.5 秒后自动隐藏 |
| 不靠近宝箱按 `E` 后立刻走进宝箱范围 | 延迟结束后不会清掉新的“按 E 打开宝箱”提示 |
| 靠近宝箱 | HUD 显示“按 E 打开宝箱” |
| 离开宝箱 | 交互提示隐藏 |
| 敌人未死亡时开宝箱 | HUD 显示“请先击败遗迹守卫” |
| 击败敌人后开宝箱 | HUD 居中显示“试炼完成” |
| 宝箱打开后再次按 `E` | HUD 显示“宝箱已经打开” |
| 玩家血量归零 | HUD 居中显示“你已倒下” |
